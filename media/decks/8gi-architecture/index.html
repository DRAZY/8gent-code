The Dual-Agent Model

Claude Code (cloud) + 8gent (local) - complementary, not competing

Claude Code

Anthropic - Cloud

  • Claude Opus / Sonnet models
  • MCP tools + Skills system
  • Complex reasoning, large context
  • Architectural decisions, security review
  • Session-scoped memory (no persistence)

8gent

Local / Free

  • Ollama local + OpenRouter free
  • Memory store + Policy engine
  • Fast iteration, autonomous loops
  • Routine tasks, factory pipeline
  • Persistent memory (SQLite, survives restarts)

Shared Workspace

Both agents read the same skills (~/.claude/skills/), operate on the same project codebases, and respect the same NemoClaw policies. They never write to the same files simultaneously. Git branch isolation keeps them out of each other's way.

Factory Pipeline

How usage patterns feed back to the collective - anonymised, opt-in, privacy-first

1
Session - member codes with 8gent, generates tool calls, memory writes, patterns
↓
2
Pattern Extractor LOCAL - extracts categories, success rates, model performance. No filenames, no code.
↓
3
Weekly Digest OPT-IN - random UUID, +/-5% noise, k>50 threshold. HTTPS POST to aggregator.
↓
4
Aggregator Fly.io AMS - write-only endpoint. No auth, no IP logging, no session cookies.
↓
5
Factory Discovery - aggregated patterns become one of 6 input sources for the AAIP pipeline.

Never transmitted: file paths, code snippets, git diffs, memory contents, credentials, IP addresses, personal identifiers.

Quarantine PR Pipeline

Every line of LLM-generated code goes through 6 gates before it reaches main

1
Local Validation - NemoClaw policy gate, TypeScript syntax, line count (max 250)
↓
2
Quarantine Branch - quarantine/{member}/{name} pushed to origin
↓
3
Automated CI - security scan, type check, lint, test, dependency audit, CODEOWNERS
↓
4
PR Created - template enforced, labels applied, reviewers assigned, Telegram notified
↓
5
Human Review - readability, correctness, edge cases, no dead code
↓
6
Merge to Main - merge commit (preserves provenance), branch deleted, CHANGELOG updated

CI Checks (all must pass)

  • Secret scanner - leaked credentials
  • Pattern blocklist - eval(), require()
  • TypeScript compilation
  • Lint enforcement
  • Unit + integration tests
  • NemoClaw policy + line count + provenance

Branch Protection

No direct push to main. No force push. No squash merge (provenance must be traceable). Auto-merge disabled org-wide. At least 1 CODEOWNER approval required.

Post-Merge

Benchmark regression check, CHANGELOG verification, Telegram notification to circle group.

Lotus Technical Stack

Three concentric rings - from strategy to execution

RING 1 - STRATEGY
SKILL.md per Chief
Each boardroom vessel (Daniel/8EO, Rishi/CTO, Samantha/CPO, Moira/CDO, Karen/CSO) has a SKILL.md defining their domain, decision authority, and evaluation criteria. The skills are read-only at runtime - no vessel can modify another's mandate.
RING 2 - PLANNING
BMAD Phased Workflows
Every significant feature goes through BMAD phases: Business case, Market analysis, Architecture design, Development plan. Each phase has a gate - no skipping. The CPO owns the workflow, the CTO validates technical feasibility.
RING 3 - EXECUTION
Sprint Execution with Gates
Work happens in sprints with quality gates between stages. Factory pipeline generates candidates, quarantine PR pipeline validates them, human review approves them. No unvalidated code reaches main. Every merge triggers regression checks.

Polyglot Strategy

One protocol, multiple implementations - shared schema, native performance

Shared Protocol Layer

Language-agnostic, JSON over stdio/WebSocket. Same daemon protocol, same NemoClaw policy format (YAML), same memory schema (SQLite), same factory digest format, same tool interface (JSON-RPC).

Anonymisation Guarantees

  • Member ID is a random UUID - not derived from email or GitHub
  • k-anonymity: digests only sent if >50 data points (prevents fingerprinting)
  • Differential noise: +/-5% on all counts
  • Aggregator has no auth, no IP logging, no cookies - write-only
  • Any member can disable sync at any time
