Threat Model Overview

Six primary threats, ranked by severity

T1 Critical

Malicious Code Injection

Compromised agent submits a backdoored pull request containing hidden payloads or exfiltration logic.

T2 Critical

Supply Chain Poisoning

LLM generates code importing malicious or typosquatted dependencies from public registries.

T3 Critical

Secret Exfiltration

Agent reads .env files, API keys, or credentials and leaks them via network requests or logs.

T4 High

Privilege Escalation

Agent attempts to bypass NemoClaw policy engine to execute blocked operations or gain elevated access.

T5 High

Data Poisoning

Adversarial member corrupts training data, memory stores, or benchmark baselines to degrade agent quality.

T6 High

Social Engineering

Member tricks another member's agent into performing unauthorized actions through prompt manipulation.

Trust Boundaries

All LLM-generated code starts untrusted, moves through quarantine, then to trusted.

Untrusted

  • All LLM output
  • External dependencies
  • New member agents
  • Network responses
  • User-supplied prompts
  • Unverified webhooks

Quarantine

  • quarantine/* branches
  • PR review queue
  • CI pipeline output
  • Staging environment
  • Unmerged factory output
  • Pending approvals

Trusted

  • main branch
  • Released packages
  • Signed releases
  • Production deploys
  • Verified member agents
  • Audited configurations
UNTRUSTED  →  QUARANTINE  →  TRUSTED

NemoClaw Policy Engine

Deny-by-default, YAML-based. Every command is evaluated before execution.

Hard Blocks (No Bypass)

  • Secrets committed to code
  • rm -rf / and destructive ops
  • Fork bombs
  • Force push to main
  • eval() in generated code
  • Exfiltration domains
  • Crypto mining processes

Approval Required

  • .env file writes
  • Pipe output to shell
  • Destructive git operations
  • npm publish
  • Docker commands
  • Network listener creation
  • File permission changes

Explicit Allows

  • Read-only git operations
  • ls, cat, bun test
  • File reads in project scope
  • Dependency listing
  • Type checking
  • Linting
  • Local dev server start
infinite_mode: false (mandatory) audit_logging: true

Quarantine Pipeline

Six-stage gate from local to main. No shortcuts.

01 LOCAL NemoClaw gate + syntax check + line count (max 250 lines per file)
02 QUARANTINE Push to quarantine/{member}/{name} branch
03 CI Security scan, type check, lint, test suite, dependency audit
04 PR Template enforced, reviewers assigned, Telegram notification sent
05 REVIEW Human review required (James initially, then Core circle)
06 MERGE Squash merge to main, branch deleted, CHANGELOG updated
Max 500 lines per PR Max 3 open PRs per member Squash merge only

Data Isolation

Clear boundary between what stays local and what is shared.

Stays Local (never leaves machine)

  • Memory database (SQLite)
  • Audit logs
  • Session history
  • Companion state
  • Git credentials
  • .env files and secrets
  • File contents and diffs
  • Claude Code memory

Shared (opt-in only)

  • Weekly usage digest (anonymised)
  • Quarantine PRs on GitHub
  • Factory-generated code
  • Aggregate stats

Anonymisation Rules

  • Random UUID (not derived from identity)
  • k-anonymity (>50 data points)
  • Differential noise (+-5%)
  • No correlation attack vectors

Supply Chain Controls

Dependencies are a primary attack surface. Every package is gated.

Package Dependencies

  • Lock files mandatory (bun.lockb committed)
  • No install scripts (postinstall blocked)
  • Allowlisted registries only (npmjs.com, jsr.io)
  • Dependency audit runs in CI on every PR
  • New dependency requires human approval
  • Typosquat detection on all installs

Model Supply Chain

  • Local models: official Ollama library only
  • Checksums verified on download
  • Custom models: signed by CAO before promotion
  • No unverified GGUF files
  • Model registry tracks provenance
  • LoRA adapters reviewed before merge

Secret Management

No secrets in code. No secrets in git history. No exceptions.

Rules

  • No secrets in source code
  • No secrets in git history
  • .env always gitignored
  • Secrets in OS keychain or vault
  • No hardcoded tokens

Incident Response

Four priority levels. Defined SLAs. Clear escalation paths.

P0 Critical 1h SLA Secret leaked, malicious code merged, production compromised
P1 High 4h SLA Suspicious PR, policy bypass attempt, agent operating out of scope
P2 Medium 24h SLA Failed secret scan, unusual network activity, audit log anomaly

The 200-Line Rule as Security

Small code is reviewable code. Reviewable code is secure code.

200
Max lines per file

Auditability

Every file can be reviewed in under 10 minutes by a single human.

Blast Radius

One file, one capability. A compromised module cannot damage everything.

Diff Clarity

Pull requests fit on one screen. No hiding changes in 2000-line diffs.

Dependency Isolation

Small modules import less. Fewer imports means a smaller attack surface.

Replacement Cost

Any module can be rewritten from scratch in a single session. If compromised, delete and rebuild.

Onboarding Security

GTM security requirements. No rogue vessels. No open doors.

Invitation Token

  • Single-use token, expires in 24 hours
  • Tied to the vouching member's identity
  • Voucher is accountable for invitee behavior
  • Token invalidated on first use or expiry
  • Revocable by any Core circle member

Vessel Hardening

  • NemoClaw baked into vessel - not configurable by member
  • Bot tokens stored in Fly secrets only
  • Vessel authenticated against membership registry
  • No unregistered vessels can join the mesh
  • Vessel identity verified on every session start

Accountability Chain

Voucher invites → Token issued (24h) → Invitee onboards → Vessel registered → NemoClaw active → Member verified

If invitee violates policy, voucher is notified and shares responsibility for remediation.

Lotus Attack Surface

Trust escalation, factory flooding, and prompt injection risks from the audit.

A1 Critical

Trust Escalation

Ring 1 member exploits promotion criteria to reach Ring 2 privileges. Automated contributions could game velocity metrics to gain trust faster than warranted.

Mitigation:

Human-verified promotion only. Three Gates block all ring transitions.

A2 Critical

Factory Flooding

Malicious member overwhelms the factory with junk PRs to exhaust review capacity, burying real attacks in noise. Volume becomes a weapon.

Mitigation:

Max 3 open PRs per member. Rate limiting on factory output. Quality gate rejects low-signal PRs.

A3 High

Vessel Prompt Injection

Adversarial content in PRs, issues, or messages crafted to manipulate reviewing vessels into approving malicious changes or bypassing policy.

Mitigation:

NemoClaw evaluates actions, not intent. Structural policy cannot be overridden by prompt content.
