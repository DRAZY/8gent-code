The 10-Article Constitution

The rules every member and agent operates under

01
No Evil, Period
No surveillance, no exploitation, no weaponization. If it harms people, we do not build it.
02
Open Source First
All collective output is open source by default. Proprietary forks require supermajority approval.
03
Privacy Is Inviolable
Agents learn locally. Data stays on your machine. No telemetry without explicit opt-in.
04
Humans Decide, Agents Execute
Agents propose. Humans approve. No autonomous action beyond the permission boundary.
05
Merit Over Hierarchy
Contributions are evaluated on quality, not seniority. Code speaks louder than titles.
06
The 200-Line Discipline
Every module under 200 lines. Auditable, replaceable, composable. No exceptions without justification.
07
Revenue Shares Back
When the collective earns, contributors earn. Transparent allocation, quarterly distribution.
08
Security Is Non-Negotiable
All LLM-generated code gets security review. No unvalidated output reaches production.
09
Accessibility By Default
Audio docs, screen readers, voice input. If someone can not use it, it is not done.
10
Fork Freedom
Any member can fork and leave at any time, taking their contributions with them. No lock-in.

The Architecture

Six working groups, each with a clear mandate

Boardroom
Strategy
Direction, priorities, roadmap. 5 permanent AI vessels (8EO, CTO, CPO, CDO, CSO) advise human leadership on decisions.
Factory
Build
The nightly pipeline. Scrapes trending repos, npm packages, research papers. Extracts patterns. Generates abilities as PRs.
Research
Discover
Autoresearch benchmarks, model evaluation, technique exploration. Finds what works and proves it with evidence.
Quality
Security
Every PR reviewed. Every LLM-generated module validated. Security gates on all factory output. No unaudited code ships.
Creative
Design
Brand, UX, documentation, presentations. Design system governance. Accessibility standards enforcement.
Infra
Deploy
CI/CD, hosting, daemon management, Fly.io deployment, monitoring. Keeps the lights on and the builds green.

Concept Extraction Sources

We import concepts, not code. Three key influences on 8GI's architecture.

gstack

CONCEPT: role-as-SKILL.md

Every role is a self-contained skill file. A Chief is not a person with a job description - they are a SKILL.md that any agent can load. Portable, composable, replaceable.

Pattern: role portability

BMAD

CONCEPT: phased workflows

Structured phases with gates between them. Plan before building. Validate before shipping. Each phase has clear entry and exit criteria. No skipping steps.

Pattern: phase gates

Hermes Agent

CONCEPT: GEPA self-evolution

Generate, Evaluate, Promote, Apply. Agents generate improvements to themselves, evaluate them against benchmarks, promote winners, and apply them. Self-evolution with guardrails.

Pattern: self-improvement loop

The One Rule

The 200-line discipline governs every module in the collective

200
lines per module, maximum.

If it is longer than 200 lines, break it up or justify it in the PR. This is not a guideline. It is the rule.

"Import concepts, not code. Read, abstract
the pattern, rebuild in 50-200 lines inside
existing architecture."

Auditability

Any engineer can read and understand a 200-line module in one sitting. No hidden complexity.

Blast Radius

Small modules mean small failures. When something breaks, you know exactly where and why.

Diff Clarity

PRs touching small modules produce clean, reviewable diffs. No 500-line changes to parse.

Replacement Cost

Any module can be rewritten from scratch in an afternoon. No single piece becomes irreplaceable.
