# 8gent Brand Heart

## Why We Exist (Purpose)

The best AI coding agents in the world — Claude Code, Codex CLI, Gemini CLI — are cloud-locked, usage-capped, and vendor-controlled. Developers hit rate limits mid-flow, watch their code stream to someone else's servers, and pay escalating subscriptions for the privilege.

8gent exists to give developers the same agentic power — **locally, privately, and without limits.**

We are the open, local-first alternative to Claude Code. The Linux to their Windows.

## How We Do It (Principles)

- **Local First**: Runs on your machine via Ollama (offline) or OpenRouter (cloud, your choice). No vendor lock-in.
- **No Caps, No Throttling**: Your hardware, your limits. Never get cut off mid-task.
- **Model Freedom**: Use any model — Llama, Mistral, Qwen, DeepSeek, or Claude/GPT via OpenRouter. Swap anytime.
- **Structured Methodology (BMAD)**: Not just a chat interface — a disciplined planning→execution→verification loop that cloud agents don't have.
- **Self-Improving**: Autoresearch and kernel fine-tuning mean 8gent gets better over time, not just when the vendor ships an update.
- **Beautiful Terminal Experience**: Rich TUI with animations, ADHD mode, voice, kanban — because developers deserve delight, not a bare prompt.
- **Fully Open**: MIT licensed, community-driven, nothing hidden.

## What We Do (Function)

8gent is an autonomous coding agent that:
- Plans and executes complex coding tasks using BMAD methodology
- Navigates codebases at the AST/symbol level (97% more token-efficient than file-level reading)
- Runs entirely on your machine — code never leaves unless you choose cloud models
- Improves itself through autoresearch and RL fine-tuning (kernel)
- Provides a rich TUI experience that makes terminal work enjoyable
- Supports any LLM — local or cloud, open or proprietary
- Gives you everything Claude Code does, minus the caps, the cloud dependency, and the $20/mo bill

## What We're NOT

- We're not an IDE plugin (we're a standalone terminal agent)
- We're not autocomplete (we're fully autonomous)
- We're not a Claude Code wrapper (we're model-agnostic)
- We're not trying to have the best model (we're trying to have the best methodology and experience)
